# TDD gap analysis — `cave-home-cover`

- **Upstream:** OpenGarage (Apache-2.0) + ESPHome cover bindings (MIT) + Somfy RTS public RF spec (clean-room)
- **License class:** permissive
- **Port method:** hybrid (ADR-015: spec-based engine for decision logic; line-by-line adapters for OpenGarage + ESPHome; clean-room for Somfy RTS RF spec)
- **Existing cave tests (agent count):** 35
- **Missing behaviors identified:** 12 (7 strict-TDD-ready)

> Generated by the spec-based gap-analysis pass (read-only). For
> `gpl-agpl-cleanroom` upstreams no upstream source/tests were consulted —
> behaviors are derived from public protocol/spec knowledge only.

**Summary.** cave-home-cover is a 0.30-fill Phase 1 MVP implementing the cover decision engine: validated position (0..=100), five-state motion lifecycle, nine device classes with feature model, position state machine (direction inference, tilt independence, stop override, obstruction safety), and grandma-friendly EN/DE/TR labeling. 35 existing unit tests cover the engine thoroughly. Missing behaviors are all deferred to Phase 1b (vendor I/O adapters, motor calibration, core integration) and carry explicit ADR-015 dispositions. Core gaps lie in multi-cover scene coordination, command sequencing safety, dynamic feature rediscovery, and locale fallback (all Phase 1b+ or permanent per Charter §7).

## Missing behaviors

### `cover_group_simultaneous_motion` — phase-1b — ✅ TDD-ready

Coordinate simultaneous motion of multiple covers (e.g., open all garage + patio doors together without sequential delays)

- **Spec:** HA cover domain: group service targets (multiple entities accept the same command atomically); cave-home Phase 1b: multi-cover sequencing and synchronization
- **Notes:** The engine handles single covers perfectly; group atomicity/sequencing is a Phase 1b adapter concern. Test: apply same command to 5 covers, verify all reach target position at same logical time step. Already deferred per parity.manifest line 42.

### `position_feedback_correction` — phase-1b — ✅ TDD-ready

Correct cover position mid-motion when real sensor feedback shows a discrepancy from the modeled position

- **Spec:** HA cover: current_position attribute updated from sensor feedback; OpenGarage: real-time distance sensor driving position estimate; cave-home Phase 1b: position_feedback() method for live sensor corrections
- **Notes:** Engine has begin/settle_at for motor phases but no live sensor-correction loop. Phase 1b motor calibration will add this. Test: begin(Open) -> mid-motion, settle_at(80) via sensor -> expect position=80, state=Stopped. Covered by deferred motor timing (line 122).

### `position_rounding_intermediate_targets` — phase-1b — ✅ TDD-ready

Handle rounding when intermediate sensor positions do not land on integer percentages (e.g., real position is 50.7%, model must round to 0..=100 u8)

- **Spec:** HA cover: position sensor may report float; cave-home cover model: u8 position enforces integer percentage. Strategy: round-nearest-even or ceil/floor.
- **Notes:** No rounding helper exists yet. Phase 1b motor adapter will feed sensor data through rounding. Test: settle_at(Position::clamped(150)) -> expect clamped to 100; add test_rounding_strategies for round/floor/ceil. Implicit in position.rs clamped().

### `position_limit_enforcement_soft_vs_hard` — phase-1b — ✅ TDD-ready

Distinguish between soft stops (user-set position limits, e.g., 'never open beyond 80%') and hard stops (mechanical endpoints)

- **Spec:** HA cover: no native soft-stop; cave-home Charter §4 safety: user-configurable limits. Phase 1b: min/max position properties and enforcement.
- **Notes:** Not in Phase 1 scope; deferred to Phase 1b with estate/household config. Test would create Cover with min/max bounds and verify SetPosition(target) clamps to [min, max]. Core safety feature; Phase 1b.

### `obstruction_event_stream` — phase-1b — ✅ TDD-ready

Expose a stream/callback of obstruction events (when/where obstruction was detected) for logging and alerting

- **Spec:** OpenGarage: obstruction distance sensor reporting; HA cover: no native event stream; cave-home Phase 1b: event publication to household automation log
- **Notes:** Engine currently records state change (Stopped) but does not expose the event itself. Phase 1b adapter will publish obstruction events. Test: apply(Open) -> report_obstruction() -> verify event in callback. Design is needed; Phase 1b.

### `obstruction_recovery_attempt` — phase-1b — ⏸ not-yet (needs impl-deps / test-after)

Automatically retry a cover motion after an obstruction is cleared (vs. requiring a new user command)

- **Spec:** HA cover: obstruction safety + auto-recovery pattern (vendor-dependent); OpenGarage: obstacle detection and retry; cave-home Phase 1b: obstruction recovery logic and retry limit
- **Notes:** Currently report_obstruction() only halts; retry logic + state tracking belong to Phase 1b adapter. Test would require retry count, backoff, and max-attempts state. Not in scope for pure engine; requires vendor-specific retry semantics.

### `feature_dynamic_rediscovery` — phase-1b — ⏸ not-yet (needs impl-deps / test-after)

Detect and adapt when a cover's feature set changes at runtime (e.g., device reconfigured to support tilt after initial discovery)

- **Spec:** HA cover: supported_features can change via integration reload; cave-home Phase 1b: device discovery and feature update callback
- **Notes:** Cover is immutable after construction; Phase 1b discovery/integration will define the update path. Current design is sound for static device configs. No test needed for Phase 1 MVP.

### `tilt_without_position_isolation` — phase-1b — ⏸ not-yet (needs impl-deps / test-after)

Verify that covers with tilt but no set_position cannot apply SetTiltPosition if the position feature is later added dynamically

- **Spec:** HA cover: supported_features controls tilt availability independently; cave-home Phase 1b: feature rediscovery updates capabilities
- **Notes:** Current design: tilt and set_position are independent flags. No dynamic update yet. Phase 1b concern; not needed for static Phase 1 devices.

### `travel_time_estimate_from_multiple_motions` — phase-1b — ⏸ not-yet (needs impl-deps / test-after)

Calibrate estimated motor travel time by observing multiple full-travel cycles and inferring time-per-percent

- **Spec:** OpenGarage: full-travel time is calibrated; cave-home Phase 1b: live calibration from repeated open/close cycles; Charter §2.2 learning-enabled systems
- **Notes:** Phase 1b motor adapter responsibility; engine does not track timing. Test would require mutable statistics accumulation. Out of scope for pure engine MVP.

### `position_state_persistence_across_restart` — phase-1b — ⏸ not-yet (needs impl-deps / test-after)

Recover the last known position after a system restart (vs. reset to fully closed)

- **Spec:** HA cover: last_position attribute can be restored from state storage; cave-home Phase 1b: state recovery from cave-home-core StateAPI
- **Notes:** Belongs to cave-home-core integration and state storage, not the engine. Engine initializes covers closed by design. Phase 1b concern per line 127 of parity manifest.

### `locale_fallback_chain` — phase-2 — ✅ TDD-ready

Gracefully fall back to EN labels when a locale is not available (e.g., user requests IT but only EN/DE/TR are shipped)

- **Spec:** Charter §6.3: mandatory languages EN/DE/TR from M1; graceful fallback for unsupported locales; Cave Charter §7: always-latest, no legacy compat
- **Notes:** status_sentence() currently panics (implicitly via match) on unknown Lang. Should add Lang::fallback_to_en(). Test: request unsupported locale, verify returns EN equivalent. Charter defers i18n expansion; Phase 2 task.

### `command_sequence_idempotence` — phase-2 — ✅ TDD-ready

Verify that repeated identical commands (e.g., Open + Open + Open) are idempotent and do not trigger spurious motion state transitions

- **Spec:** HA cover: repeated set_position(50) on a cover already at 50 must not re-trigger motion; cave-home engine: begin() direction inference should detect no-op
- **Notes:** Partially tested: begin_to_same_position_does_not_move() covers SetPosition case. Missing: repeated Open while already Open, repeated Close while Closed, repeated Stop while Stopped. Test: apply(Open) -> apply(Open) -> verify state=Open, no spurious Opening. 3 small integration tests.
